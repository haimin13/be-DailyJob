# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
- 프로젝트 디렉토리에 index.html이 존재할 경우 자동 로드
- GET 회원가입 기능 추가
- url 파싱 유틸리티 클래스 추가
- 정적 파일 요청이 아닌 로직 요구를 핸들링 하는 인터페이스 구현
- 회원가입 요청시 파라미터 유효성 검사 추가

- 끝내지 못한 것: 테스트 코드 작성, HTTP 상태코드 적용

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: 로직이 필요한 요청핸들러 인터페이스를 만든 것
- 그렇게 판단한 이유: 기존 핸들러 클래스에 기능을 추가할 때마다 코드를 추가하면 가독성도 떨어지고, 분기 처리가 과하게 복잡해지며, 디버깅이나 확장에도 좋지 않을 것이라고 생각했다.
- 고려했지만 선택하지 않은 대안: 기존 메소드에 스파게티코드 작성
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 새로운 기능을 추가할 때 인터페이스에 맞게 구현하고 결과만 리턴하는 방식으로 유지보수가 가능해진다. 또한 추후에 테스트 코드 작성도 간단해진다는 점이 있다.

### 3. 가장 막혔던 문제와 해결 과정
- 오늘 개발하면서 막힌 부분은 없었다. 다만 개발자로서의 지식 부족을 많이 느낀 하루였다.
- 머릿속에 있는 개념을 말로 설명하는 것이 어려웠다. 구조적으로 생각하는 연습을 해야겠다.
- ERD 설계를 조원들과 같이 했는데 짠 이유가 머릿속에 있는데 그걸 조원들에게 설명하는 게 안돼서 고민하느라 시간을 많이 썼다.
- 또 WAS 과제를 조 안에서 빨리 한 편이었는데, 조원이 물어본 것에 명확하게 답변해주지 못해서 아쉬웠다.
- AI가 설명해준것을 당연하게 받아들이고 코드에 반영하기 급했는데, 왜 그런 결정을 했는지 더 깊게 파고들어 생각해보고 장점과 단점을 명확히 파악하고 설명하는 능력이 앞으로 협업하는 데도 중요할 것이라고 생각한다.

### 4. 오늘 잘한 행동(개발과 관련된 것)
- DA 수업에서 그동안 놓쳤던 부분을 배웠다.
- 슈퍼타입-서브타입 / 모델링 단계 / 좋은 ERD 설계 / 까마귀발 / FK 식별-비식별 관계 / M:N 관계 해소- 교차테이블
- ERD 설계 하면서 기수성이 많이 헷갈렸다.
- JVM에서 static의 역할: 서버 처음 실행할 때 한 번 실행되고 메서드 영역에 저장되어 모든 스레드가 공유한다. 동시성 문제가 생길 수 있다.
- 자바의 거의 모든 객체는 힙 영역에 생성된다.
- 참조 변수가 스택 영역에 생성되어 특정 스레드만 사용할 수 있다. 리모컨 느낌.
- String을 응답에 주고 싶으면 인코딩해서 .getBytes()를 이용해 byte[]로 변환해야 한다.

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 내가 쓴 코드의 구조를 머리속에 명확히 이해하고 설명하는 능력 부족
- 코드 짜기 전에 이유를 생각하고 리서치를 많이 해야 명확하게 머리속에 남을 것 같다.
- 공부한 것은 바로바로 정리하는 습관 기르기
- 기술적으로 깊게 파고들기
