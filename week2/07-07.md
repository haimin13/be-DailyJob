> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - HttpResponse 클래스에 정적 팩토리 메소드 추가 구현
  - 쿼리 파라미터 파싱 정책 변경
  - POST 로그인 구현
    - UserLoginHandler 클래스 생성
    - 세션아이디를 ThreadLocalRandom을 이용해 생성
    - HttpResponse 클래스의 bakeCookie 함수 구현: 쿠키 값 조립 및 객체에 태움.
  - BadRequest 응답 고도화 
    - script를 포함한 html을 응답으로 전달해 사용자가 alert로 에러 내용을 확인하고 history.back()을 이용해 원래 페이지로 돌아가도록 구현
    - user/login과 user/create에 적용됨.
  - 로그아웃 구현
    - Set-Cookie의 sid 값을 비우고 Max-Age값을 0으로 설정해 브라우저의 sid를 무효화
    - 성공시 /index.html로 리다이렉트.
    - /main/index.html의 로그아웃 버튼을 form태그로 감싸 POST /user/logout 호출하도록 수정
  - 회원가입 조건 체크 강화: 4글자 이상 제약 추가.
  - 로그 간소화
  - 학습:
    - HTTP Url에는 코드에서 처리한 것 외에도 #(fragment)라는 게 있다. 프론트엔드에서 데이터의 특정 위치를 파악하기 위해 작성하는 것이다. 서버에는 전달되지 않는다.
    - 메소드 이름은 this.getClass().getSimpleName()으로 가져올 수 있다. 로깅에 필요할지도.
    - Java에서 무작위로 값을 생성하는 방법은 여러개 인데, Random과 ThreadLocalRandom의 차이에 대해 학습헀다. 간단하게 후자는 각 스레드별로 고유하게 갖고 있는 거라 다중스레드 환경에서 중복이 발생하지 않는다는 것이다.
    - Set-Cookie로 어떻게 쿠키값을 생성하고 세션을 관리하는지 학습했다. Path로 쿠키 적용 범위를 지정할 수 있고, Max-Age로 유효기간을 설정할 수 있다. 0으로 설정하면 즉시 파기된다.
    - StringBuilder는 반복문에서나 쓰는 거다.
    - 재로그인, 로그아웃시 쿠키를 어떻게 처리해야 하는지. 브라우저가 알아서 overwrite하긴 한다. 그래도 처리해야죠?
    - 톰캣은 왜 요청 당 스레드를 생성했는가.
      - 개발의 편의성을 위해 Trade-off -> 쓰레드풀로 영끌 -> 가상메모리 등장.
    - 서블릿이란? 
      - 자바 기반 웹 요청 처리 및 동적 콘텐츠를 생성하는 서버측 표준 기술.
      - 서블릿 컨테이너(WAS)가 통신관리, 서블릿 생명주기 관리, 멀티스레딩 관리를 해줌
      - 현대 자 웹개발에서 스프링 MVC의 핵심인 DispatcherServlet이 중앙에서 처리함.
- (오늘의 목표 중에서) 아직 끝내지 못한 부분:
  - 세션 아이디 암호화해서 생성
  - 쿠키 값 보관 필요
  - 포스트맨 사용법 익히기

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: BadRequest 응답을 정적 팩토리 메소드로 생성해서 html형식으로 제공
- 그렇게 판단한 이유: 유저가 이유를 알아야 고쳐서 다시 요청하니까
- 고려했지만 선택하지 않은 대안: 헤더에만 담아서 유저가 개발자도구 켜서 확인해야 알 수 있음
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 회원가입, 로그인 뿐만 아니라 모든 요청에 대해서 에러 이유만 적으면 자동으로 응답 생성할 수 있어서 확장이 편할 것 같다.

### 3. 가장 막혔던 문제와 해결 과정
- 쿠키 헤더 작성할 때 문자열을 잘못 적어서 잘못된 쿠키 값이 20분동안 남아있어서 디버깅이 힘들었다.
- 사파리 말고 크롬으로 다시켜서 해결했다.
- 또한 로그가 너무 많이 나와서 디버깅에 필요한 로그 찾기가 힘들었다. 그래서 최대한 로그를 간결하게 바꿨다.
