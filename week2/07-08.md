### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: HttpRequest 클래스 생성 및 코드 리팩토링
- 그렇게 판단한 이유: requestHandler에서 파싱을 하고 지역변수로만 결과를 갖고 있었는데, 메소드에서 사용하려면 일일이 인자로 던져주는 게 귀찮았고 관리도 어려웠다.
- 고려했지만 선택하지 않은 대안: RequestHandler에서 지역변수로 두기
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향:
  - HttpRequest 생성할 때 파싱을 다 해서 필드값으로 저장하고, 메소드 호출할 때 객체만 넘겨주면 되어 매개변수 실수 할 여지를 제거했다.
  - 헤더 파싱 관련 함수를 다 클래스 메소드로 옮겨서 응집도를 높였다.

### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: 로그아웃 할 때 세션 만료 기능이 동작을 안함 (DB에서 sessionId를 찾을 수 없음)
- 처음 추측한 원인: SessionManager.expireSession 함수에 String 인자가 null로 들어와서 그럴 것이다.
- 원인을 확인하기 위해 시도한 방법:
  - 핸들러에서 getCookies.get("sessionId")으로 쿠키값 가져올 때 null을 빈문자열로 변환
  - expireSession 함수에 디버깅 코드 추가 (DB 조회, 쿠키값 조회)
  - claude에게 로그와 함께 코드를 실행해서 원인 파악하게 시도
- 실제 원인:
  - 요청 쿠키값 파싱할 때 key와 value에 앞에 공백이 포함되어 있어서 sessionId와 일치하는 키를 못 찾음.
  - 디버깅 코드를 봐도 공백이 있는 지 없는지 확인하기 어려웠다.
  - claude가 원인을 발견했다.
- 해결 방법: key와 value에 .strip()으로 공백 처리
