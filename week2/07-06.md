# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
#### 개발
- POST 회원가입 구현
  - Request Body byte값 읽기
  - userId 중복 체크
  - 가입 완료 후 리다이렉트
  - 파라미터 읽을 때 NPE 방지 및 예외 처리
- HttpResponse 클래스 고도화
  - 응답헤더 리스트
  - 정적 팩토리 메소드 구현 (200, 302)
  - 객체 생성 시 Content-Length 헤더 값 계산
- Request Header 파싱 및 로깅 고도화
- 리팩토링
  - 정적 파일 서빙하는 StaticFileRequestHandler
  - 헤더 생성 책임 분리 (각 핸들러에서 처리)
#### 학습
- Java 단락 평가, throw(메서도 호출한 상위 계층으로 에러 패스)
- 정적 팩토리 메서드
- 관심사 분리, 단일 책임 원칙
- auto scaling, L4·L7 로드밸런서, PUT과 PATCH의 차이.
- sticky session, JWT가 어떻게 stateless하게 유저 정보를 관리하는지
- request header, referer
- 응답 출력은 마지막에 조립해서 하기.

- (오늘의 목표 중에서) 아직 끝내지 못한 부분:
- 학습: Optional, Exceptions, BufferedReader, InputStream, Record
- 개인적으로 <주니어 백엔드 개발자가 반드시 알아야 할 실무 지식>이라는 책을 읽고 있는데 오늘 한 챕터도 못 읽었다. 내일 반납인데 큰일 났다.
- 개발: Multipart 처리, 라우팅 고도화, 예외 처리 고도화, POST 응답 헤더 Date 추가

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단: HttpResponse 클래스 고도화
- 그렇게 판단한 이유: 여러 타입 요청에도 공통된 코드를 재사용할 수 있게.
- 고려했지만 선택하지 않은 대안: 콘텐츠 헤더를 핸들러에서 따로 관리.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 새 라우트에 대해서 중복되는 코드를 작성할 필요가 없어짐.

### 3. 가장 막혔던 문제와 해결 과정
- 아직은 크게 막히는 부분은 없었다.
- 다만 좀 더 효율적으로 할 수 있는 부분은 있는 것 같다.
- 구체적으로 무엇인지 알아가자.
- + claude 설치했는데 좋은 기능이 많은 것 같다. 근데 실제로 유용하게 쓰려면 숙지가 필요할 듯 하다.

### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: Http spec 문서 찾아보면서 코드 작성한 것. 미래를 고려하여 body 읽는 코드 짠 것. POST를 위해 body를 파싱해야 하는데, 4단계 요구사항은 string parameter지만 바이너리 파일을 주고 받을 수도 있기 때문에 byte[] 형식으로 받도록 했다.
- 그 접근이 도움이 된 이유: 핸들러에서 용도에 맞게 변환하는 식으로 코드를 짰기 떄문에 나중의 수고를 덜었다고 생각한다.
- 다음에도 반복하고 싶은 점: 지금하는 선택이 나중에 어떤 영향을 미칠지 고려하며 개발하기.

### 5. 오늘 아쉬웠던 행동(개발과 관련된 것)
- 비효율적이었던 접근/놓치거나 부족했던 점: Java 문법과 내장 함수를 계속 찾아보면서 작성함.
- 그렇게 된 원인: Java에 익숙하지 않아서.
- 다음에는 다르게 해볼 구체적인 행동: 한 번 써본 함수는 외우기.
